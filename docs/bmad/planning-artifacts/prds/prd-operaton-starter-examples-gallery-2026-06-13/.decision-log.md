# Decision Log — Examples Gallery PRD

## 2026-06-13 — Activation
- Intent: Create. Stakes: internal/project-team. Mode: Fast path.

## 2026-06-13 — Discovery decisions
- **Registry scope (v1):** Maintainer-controlled hard-coded list; future-extensible. *Why:* avoid premature config surface; ship value first. *How to apply:* repo list lives in `StarterProperties`/`application.properties` only.
- **Download flow:** Stream example as ZIP. *Why:* matches existing GenerateController UX; "immediately downloaded" requirement. *How to apply:* new endpoint that fetches example subfolder from GitHub and returns ZIP.
- **Repo shape:** Monorepo with subfolders; manifest lists examples with `path:` per entry. *Why:* aligns with `kthoms/operaton-examples`; one HTTP fetch per repo.
- **Fetch policy:** At server startup only. *Why:* simplest, no caching infra, restart-to-refresh is acceptable for an internal-stakes feature.
- **Failure mode:** Log + skip silently; surface what loaded. *Why:* resilient UX; one bad repo shouldn't break the gallery.
- **Coexistence:** Two subsections within one Gallery page (Examples + Use Cases). *Why:* smooth migration path; use cases not yet retired.

## 2026-06-13 — Post-draft adjustments
- **Manual refresh in v1:** added `POST /api/v1/examples/refresh` (FR-B9). *Why:* avoid restart-to-refresh friction now that the surface is mostly built anyway. Auth deferred (Open Q-1).
- **Ref pinning:** loader resolves `@ref` → commit SHA at each load; all manifest + download fetches use the pinned SHA. *Why:* keep manifest view and downloaded code consistent.
- **Server-side ZIP cache:** keyed by `(owner, repo, sha, exampleId)`; LRU eviction with `maxSizeMb` cap. *Why:* lower latency on repeat downloads, decouple from GitHub availability after first build; SHA-keying gives natural invalidation on refresh.
- **Optional icon field:** emoji or repo-relative image path, rendered on cards. *Why:* visual scannability; user-requested.

## 2026-06-15 — Use Cases Deprecation
- **Change:** Remove Use Cases feature from operaton-starter. Built-in use cases have been migrated to `kthoms/operaton-examples/use-cases`.
- **Scope:** Updated FR-D1 (only Examples subsection now), FR-D3 (search filters Examples only), FR-D4 (filter chips apply to Examples only), phasing (removed v2 use-case migration task), non-goals (added deprecation note).
- **Impact:** Gallery now focused on Examples. Project Types remain unchanged. Search and filters simplified.
- **Decision:** Simplify gallery to Examples-only; omit links to external use-cases repo from docs.

## 2026-06-15 — UI Refinements & Error Handling (v1)
- **Button label change:** "Download ZIP" → "Download". *Why:* shorter, clearer action label. *How to apply:* update FR-D2, FR-D6.
- **Detail expand affordance:** Add explicit `?` icon to toggle detail expansion. *Why:* improves discoverability; aligns with Project Types behavior. *How to apply:* FR-D2 specifies the icon; update ExampleGalleryCard.vue.
- **Tags in details:** Tag chips now appear in both card summary and expanded detail view. *Why:* full context visible when detail is expanded; consistent with card display. *How to apply:* FR-D2 updated; pass tag list to detail component.
- **Missing example handling:** Examples that don't exist at the pinned SHA receive a disabled Download button + warning icon (⚠️) with link to source repo. *Why:* gracefully surfaces failures (500 NoResourceFoundException bug); lets users report broken links rather than hitting error page. *How to apply:* FR-C1 adds `isDownloadable` field; FR-C5 validates examples on load; FR-D6 disables button + shows warning icon.
- **Validation strategy:** Async HEAD-request validation on manifest load (startup, refresh), non-blocking; `isDownloadable` updated in background. *Why:* keeps load fast; reflects current state without slowing down startup. *How to apply:* FR-C5 new, describes async validation. Rate-limit caveat added to R-1.
- **Backend error handling:** Download endpoint properly routed to avoid static resource handler; returns 404 with structured body if example missing. *Why:* fixes 500 NoResourceFoundException bug where request was falling through to static handler. *How to apply:* FR-C2, FR-C3 updated; ensure route config maps `/api/v1/examples/*` before static pattern.
