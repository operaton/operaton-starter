## 2026-06-13 — Update: Examples Gallery feature

Linked PRD: `../../prds/prd-operaton-starter-examples-gallery-2026-06-13/prd.md`. Mode: Update + Fast path.

- **Scope.** Add an "Examples" subsection to GalleryView alongside (not replacing) the existing Use Cases. Inherits all existing tokens; no new colors.
- **Search & filter placement.** Sticky bar above both subsections — one shared search input + filter chip row. Chips (runtime / buildSystem / complexity / integrations) apply to Examples only; search applies to both. Matches `code.quarkus.io` pattern.
- **Card density.** Compact card, expand-on-click for detail metadata. Reuses HelpIcon accordion pattern. Anatomy: icon + title + shortDescription + three metadata badges + tag chips + Download ZIP + View on GitHub + collapsed details panel.
- **Download UX.** Primary button on card. States: idle → "Downloading…" + disabled + spinner → transient "Downloaded {exampleId}.zip" ({colors.success}) for ~3s → idle. Failures use card-local inline error styled like `{components.error-banner}`, not the global ErrorBanner — failures are scoped per example.
- **No refresh UI in v1.** Refresh endpoint is API-only. No `/diagnostics` route, no header refresh icon.
- **Icon rendering.** Emoji as plain text in a sized container; image paths fetched from GitHub raw URL at the pinned SHA; fallback to a generic glyph when absent or load fails.
- **Empty-state copy.** No examples load → lead with the "what" not the "how"; link to format docs. Empty filter results → "No examples match these filters" + "Clear filters" ghost button.

### Post-review fixes (rubric walker, 2026-06-13)

- **Subsection order:** Project Types stays first (preserves Marcus/Elena/Priya flows in §8). Examples + Use Cases sit below a sticky GallerySearchBar that is scoped to those two subsections only. Search does not affect Project Types.
- **MetadataBadge as Tag variant:** complies with PRD FR-D7. The unified `Tag` model is reused; visual lanes are driven by `TagCategory` via `tagColors.ts`. `runtime`/`buildSystem`/`complexity` route to the monochrome `{components.metadata-badge}` rendering; `concept`/`integration` keep the accent `{components.tag}` rendering.
- **Filter chip ARIA:** changed `role="switch"`/`aria-checked` to `aria-pressed` toggle buttons (ARIA APG idiomatic for filter pills).
- **MED/LOW findings:** rolled forward as implementation-time concerns (concurrent download semantics, pluralization edge cases in the live region, desktop-chip touch target). See `review-examples-gallery-rubric.md`.
